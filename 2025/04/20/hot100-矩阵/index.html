给定一个 m x n 的矩阵，如果一个元素为 0 ，则将其所在行和列的所有元素都设为 0 。请使用 原地 算法。

示例 2：

提示：

  • m == matrix.length
  • n == matrix[0].length
  • 1 <= m, n <= 200
  • -2^31 <= matrix[i][j] <= 2^31 - 1

进阶：

  • 一个直观的解决方案是使用 O(mn) 的额外空间，但这并不是一个好的解决方案。
  • 一个简单的改进方案是使用 O(m + n) 的额外空间，但这仍然不是最好的解决方案。
  • 你能想出一个仅使用常量空间的解决方案吗？

思路

用矩阵的第一行和第一列标记每行和每列是否出现 0。但这样会导致原数组的第一行和第一列被修改，无法记录它们是否原本包含 0。因此我们需要额外使用两个标记变量分别记录第一行和第一列是否原本包含 0。

题目

48. 旋转图像 - 力扣（LeetCode）

给定一个 n × n 的二维矩阵 matrix 表示一个图像。请你将图像顺时针旋转 90 度。

你必须在 原地 旋转图像，这意味着你需要直接修改输入的二维矩阵。请不要 使用另一个矩阵来旋转图像。

示例 1：

示例 2：

提示：

  • n == matrix.length == matrix[i].length

  • 1 <= n <= 20

  • -1000 <= matrix[i][j] <= 1000

思路

非常好题解：48. 旋转图像 - 力扣（LeetCode）

核心就是找到旋转前后的坐标关系：matrix[i][j] = matrix[n - 1 - j][i];

只要有这一个就能写出所有的，可以把 i 和 j 看做变量，指代任何东西，写第二个公式 matrix[n - 1 - j][i] = matrix[n - 1 - i][n - 1 - j] 的时候，只需用 n - 1 - j 代换 i，用 i 代换 j即可，以此类推。

只用以左上角的矩阵为起始点即可，因为每次旋转是四个一组。

240. 搜索二维矩阵 II - 力扣（LeetCode）

编写一个高效的算法来搜索 m x n 矩阵 matrix 中的一个目标值 target 。该矩阵具有以下特性：

  • 每行的元素从左到右升序排列。
  • 每列的元素从上到下升序排列。

示例 1：

示例 2：

提示：

  • m == matrix.length
  • n == matrix[i].length
  • 1 <= n, m <= 300
  • -1e9 <= matrix[i][j] <= 1e9
  • 每行的所有元素从左到右升序排列
  • 每列的所有元素从上到下升序排列
  • -1e9 <= target <= 1e9

思路

从右上角开始寻找元素。

  1. 当前元素比目标值小，向下寻找
  2. 当前元素比目标值大，向左寻找
  3. 当前元素与目标值相同，结束寻找

思考：为什么要从右上角开始搜索？

如果从左上角开始搜索，右边和下面的元素都比它大，不知道该向哪搜索。右上角可以唯一确定搜索方向，每次排除掉一整行或一整列的数。同理，从左下角开始搜索也是可以的。
